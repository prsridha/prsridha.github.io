Research

My research interests lie at the intersection of Machine Learning and Software Systems (i.e. ML Systems). I develop algorithms, build platforms and infrastructure needed to improve the performance and scalability of end-to-end Machine Learning pipelines. In doing so, I hope to improve the productivity of Data Scientists and ML Engineers, and at the same time, make machine learning more accescibile to the everyday user by reducing time and costs. My expertise lie in software systems, cloud computing, distributed computing, containerization and container orchestration. I have deep technical know-how and hands on exprience with today's Machine Learning algorithms, as well as their inner workings in the context of its scalability and "distributability".

Current Work

Cerebro - a distributed deep learning framework that offers effecient model selection pipelines on multi-node multi-GPU clusters. Two existing (and growing) execution strategies developed in our group are - Model Hopper Parallism which is a hybrid of task and data parallism, and Hydra - which focuses on model sharding. My work involves designing a unified scheduler, along with the surrounding infrastructure, that can work across all three strategies - model parallism, data parallism and task parallism, in an effecient manner. This ensures that small deep learning models that work on large data - multimedia such as images and videos , as well as large models that don't fit on a single GPU's memory - such as Transformers, can make use of Cerebro's increased performance and scalability. Current experiments are on 16-GPU 4 node/8node AWS clusters, using Dask for the ETL pipeline and Kubernetes for container orchestration.
